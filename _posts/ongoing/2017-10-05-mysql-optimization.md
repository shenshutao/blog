# 概述
数据库的性能基于数据库层的几个因素，比如表，查询和配置。当然这些软件上的操作最终会落实到硬件层的CPU和IO操作。所以Mysql的数据库优化包括以下方面：
- 数据库层优化
- 硬件层优化
- 平衡便携性和性能


# 数据库层优化
- 表设计优化， 列使用正确的类型    
- 索引的优化
- 正确选择存储引擎 InnoDB / MyISAM
- 数据存储格式 （压缩）优化
- MySQL的锁策略优化
- 缓存配置优化


# 硬件层优化
- 磁盘搜索/寻道速度
- 磁盘读写速度
- CPU周期
- 内存带宽
